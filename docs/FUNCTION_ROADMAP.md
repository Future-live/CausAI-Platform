# CausAI Platform 功能扩展设计

本文档整理当前数据分析系统后续可以实现的功能，并按价值、实现复杂度和与现有架构的契合度进行设计。

## 当前能力基线

系统已经具备：

- 用户认证与权限隔离。
- 多格式表格数据上传。
- 数据集与数据版本管理。
- 缺失值处理、筛选、字段启用/禁用。
- 字段画像、字段分布、数据预览。
- 拖拽式统计图表配置与图表生成。
- PC/GIES 因果发现任务。
- 因果图、结果表和效应指标。
- 因果任务进度、worker 标识、取消与重试。
- 图表推荐、相关矩阵、分组聚合、假设检验、异常值检测。
- PC/GIES 参数化入口，其中 PC 支持 `algorithm_params.alpha`。
- DeepSeek 语义辅助分析。
- 收藏夹，支持类型、数据集、分组、关键词、时间范围筛选，以及快照保存。
- 可复现分析工作流，支持保存并运行数据准备、筛选和字段启用步骤。
- 标准 `/api/v1` 调用入口。

## P0：应优先完善的生产化能力

### 1. API Key 认证

现状：

- 外部调用需要先登录获取 JWT。

设计：

- 新增 `api_keys` 表：`id`、`owner_id`、`name`、`key_hash`、`prefix`、`scopes_json`、`last_used_at`、`expires_at`、`is_active`。
- 用户在个人信息页生成和吊销 API Key。
- `get_current_user` 增加 `X-API-Key` 解析逻辑。
- scope 示例：`datasets:read`、`datasets:write`、`causal:run`、`llm:use`。

价值：

- 更适合长期服务集成。
- 不需要把用户名密码交给第三方系统。

### 2. 独立异步任务队列

现状：

- 当前已有任务进度、`started_at`、`worker_id`、取消与重试。
- FastAPI `BackgroundTasks` 可运行因果任务，但仍不适合大量并发和长任务。

设计：

- 引入 Celery/RQ/Arq 其中一种轻量队列。
- 将大文件画像、因果分析、LLM 分析迁移到独立 worker。
- 页面可继续通过轮询任务详情展示进度，后续可追加 Server-Sent Events。

价值：

- 防止请求线程被长任务占用。
- 支持任务重试、取消和失败排查。

### 3. 文件导入质量报告

设计：

- 上传后生成 `DatasetQualityReport`：
  - 行列数。
  - 缺失率。
  - 重复行数。
  - 常量列。
  - 高基数字段。
  - 异常值比例。
  - 字段类型推断置信度。
- 存入 `dataset_versions.operations_json` 或单独 `dataset_quality_reports` 表。
- 上传页和数据准备页展示导入质量摘要。

价值：

- 数据分析平台的第一步应该帮助用户判断数据是否可靠。
- 能显著提升数据准备页的专业感。

## P1：提升分析能力的功能

### 4. 因果算法扩展

可加入算法：

- FCI：支持潜在混杂。
- LiNGAM：线性非高斯场景。
- NOTEARS：连续优化 DAG 学习。
- Granger Causality：时间序列因果。
- Double ML / DoWhy：因果效应估计。

设计：

- `AnalysisJob.algorithm` 继续作为统一入口。
- `algorithms/` 下按算法拆模块。
- `CausalJobCreate` 已增加 `algorithm_params`，后续继续扩展具体算法参数。
- 前端按算法动态展示参数面板。

价值：

- 平台定位会从简单因果发现升级为因果分析工作台。

### 5. 背景知识管理

设计：

- 新增 `knowledge_sets` 表：
  - 名称。
  - 变量集合。
  - 必须边。
  - 禁止边。
  - 说明。
- 因果分析页允许保存当前背景知识图。
- 后续任务可以复用背景知识模板。

价值：

- 真实业务分析通常需要复用领域知识。
- 减少重复配置。

## P2：增强协作和沉淀的功能

### 6. 分析报告生成

设计：

- 用户从收藏夹选择图表、因果图、语义解释。
- 生成 HTML/PDF 报告。
- 报告内容包括数据集版本、操作链路、图表、结论和复现参数。
- 新增 `reports` 表和 `/api/v1/reports`。

价值：

- 分析结果从“页面上看”变成“可交付文档”。

### 7. 多用户协作

设计：

- 新增 workspace/project 概念。
- 数据集、任务、收藏、报告绑定 workspace。
- 权限分为 owner、editor、viewer。
- 操作日志记录数据上传、删除、任务运行、报告导出。

价值：

- 从个人工具升级为团队分析平台。

### 8. 数据源连接器

可支持：

- PostgreSQL/MySQL。
- S3/OSS/COS。
- HTTP 文件 URL。
- Google Sheets/Excel Online。

设计：

- 新增 `data_sources` 表。
- 导入任务统一为 `ingestion_jobs`。
- 文件上传只是 ingestion 的一种 source_type。

价值：

- 让平台能够接入真实企业数据源。

## UI/UX 后续优化建议

### 数据上传页

- 上传后展示字段画像摘要。
- 历史数据增加格式、行列数、最近使用时间。

### 数据准备页

- 增加操作历史时间线。
- 支持撤销到任意数据版本。
- 缺失值处理提供字段级策略。

### 统计分析页

- 支持多图布局和图表配置保存。
- 图表导出支持 PNG、SVG、CSV、JSON。

### 因果分析页

- 背景知识图支持禁止边、必须边、无向边标记。
- 结果图支持节点搜索、边过滤和布局切换。

### 语义理解因果页

- 增加“分析上下文”摘要。
- LLM 输出结构化：结论、依据、风险、下一步建议。
- 支持一键保存为报告段落。

## 推荐实施顺序

1. API Key 认证。
2. 导入质量报告。
3. 独立异步任务队列。
4. 报告生成。
5. 因果算法扩展。
6. 团队协作和数据源连接器。
